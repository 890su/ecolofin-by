# Инструкция по загрузке на хостинг

## Что загружать на хостинг

После выполнения `npm run build` все файлы для загрузки находятся в папке **`dist/`**

## Шаги для загрузки:

### 1. Соберите проект
```bash
npm run build
```

### 2. Содержимое папки `dist/` нужно загрузить на хостинг

Структура папки `dist/`:
```
dist/
├── .htaccess          ← Файл конфигурации Apache (уже включен)
├── index.html         ← Главная страница
├── privacy/
│   └── index.html     ← Страница политики конфиденциальности
├── img/               ← Все изображения
├── robots.txt         ← SEO файл
├── sitemap.xml        ← SEO файл
└── favicon.ico        ← Иконка сайта
```

### 3. Куда загружать на хостинге

**Обычно это одна из папок:**
- `public_html/`
- `www/`
- `httpdocs/`
- `htdocs/`
- Корневая папка домена

**Уточните у вашего хостинг-провайдера**, какая папка используется для вашего домена.

### 4. Как загружать

**Вариант 1: FTP/SFTP**
- Используйте FileZilla, WinSCP или другой FTP-клиент
- Подключитесь к хостингу
- Загрузите ВСЕ содержимое папки `dist/` в папку домена на хостинге

**Вариант 2: Панель управления хостингом**
- Войдите в панель управления (cPanel, Plesk и т.д.)
- Откройте файловый менеджер
- Загрузите все файлы из `dist/`

**Вариант 3: Git (если поддерживается)**
- Если хостинг поддерживает Git, можно настроить автоматический деплой

### 5. Важные файлы

✅ **`.htaccess`** - уже включен в сборку, обеспечивает:
- Редирект на HTTPS
- Безопасность
- Сжатие файлов
- Кэширование
- Защиту от спама

✅ **`robots.txt`** - для поисковых систем

✅ **`sitemap.xml`** - карта сайта для SEO

### 6. После загрузки

1. Проверьте сайт: откройте `https://ecolofin.by` в браузере
2. Проверьте, что HTTPS работает (должен быть редирект с HTTP)
3. Проверьте формы - они будут показывать контакты (fallback), так как API endpoints не работают на статическом хостинге

### 7. Для работы форм

Если нужно, чтобы формы отправлялись в Telegram:

**Вариант А: Vercel/Netlify (рекомендуется)**
- Бесплатно
- Поддерживают serverless функции
- API endpoints будут работать

**Вариант Б: Внешний сервис**
- Formspree, EmailJS и т.д.
- Нужно обновить код форм

**Вариант В: Хостинг с Node.js**
- Нужен адаптер `@astrojs/node`
- API endpoints будут работать

## Проверка после загрузки

1. ✅ Главная страница открывается
2. ✅ Все изображения загружаются
3. ✅ Стили применяются
4. ✅ Формы показывают контакты (fallback)
5. ✅ HTTPS работает
6. ✅ Мобильная версия работает

## Если что-то не работает

- Проверьте права доступа к файлам (обычно 644 для файлов, 755 для папок)
- Проверьте, что `.htaccess` загружен
- Проверьте логи ошибок в панели хостинга
- Убедитесь, что все файлы загружены (особенно из папки `img/`)
