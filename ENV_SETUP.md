# Настройка .env файла

## Где находится файл .env

Файл `.env` находится в **корне проекта** (там же, где `package.json`, `astro.config.mjs`):

```
ecolofin-by/
├── .env              ← ВОТ ОН ЗДЕСЬ
├── package.json
├── astro.config.mjs
├── src/
└── dist/
```

## Как отредактировать .env

### Вариант 1: Через редактор кода (VS Code, Cursor)
1. Откройте проект в редакторе
2. Найдите файл `.env` в корне проекта
3. Откройте и отредактируйте

### Вариант 2: Через Блокнот (Notepad)
1. Откройте Блокнот
2. Файл → Открыть
3. Перейдите в папку `D:\aibrain\04_projects\ecolofin-by`
4. В типе файлов выберите "Все файлы (*.*)"
5. Откройте файл `.env`
6. Отредактируйте и сохраните

### Вариант 3: Через PowerShell
```powershell
notepad .env
```

## Важно

- ✅ Файл `.env` уже заполнен токенами
- ✅ Он находится в корне проекта
- ✅ НЕ нужно копировать его в `dist/` (для статического хостинга не нужен)
- ✅ Он нужен только для разработки (`npm run dev`) и для serverless функций

## Для деплоя на Vercel/Netlify

Если используете Vercel или Netlify, переменные окружения настраиваются в их панели управления:

1. Зайдите в настройки проекта
2. Найдите "Environment Variables" или "Переменные окружения"
3. Добавьте:
   - `TELEGRAM_BOT_TOKEN` = `8503860004:AAHjB6l5VJ2D9NP8oGd8gGuczSNmH5QP9u8`
   - `TELEGRAM_CHAT_ID` = `-5240163266`

## Проверка

Токены уже записаны в файл. Можно проверить:
```bash
# В PowerShell
Get-Content .env
```

Или просто откройте файл `.env` в редакторе.
